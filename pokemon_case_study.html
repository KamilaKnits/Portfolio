Pokemon - Case Study

Overview

I built a small web application using HTML, CSS, and JavaScript that pulls data from an external API and displays it in a clean, interactive interface. The goal was to practice working with real data, strengthen my front‑end fundamentals, and create a polished project I could confidently showcase in my portfolio.

Purpose and Context

As part of leveling up my JavaScript skills, I wanted to build a complete, end‑to‑end project—not just isolated exercises. Since I wasn’t creating my own API yet, I chose to work with a public data source and focus on building a smooth, user‑friendly experience around it.

This project also gave me space to refine the full front‑end stack: semantic HTML, responsive CSS, and clean, well‑structured JavaScript. Beyond functionality, I wanted the app to feel intentional and visually cohesive, not like a rough prototype.

Objective

Create a functional, aesthetically consistent web app that:

  • Loads data from an external API
  • Displays a list of items
  • Allows users to click an item to view more detailed information

The project needed to be error‑free, easy to navigate, and deployed publicly.

Step 1: Defined the Data Source

I selected a public API and explored its endpoints to understand what data I could use and how it was structured.

Step 2: Set Up the Project Structure

I created the base HTML, linked my CSS and JavaScript files, and set up a clean folder structure to keep the project organized.

Step 3: Fetched and Stored API Data

Using fetch(), I loaded data from the API, handled responses, and stored the results in a JavaScript array for easy access throughout the app.

Step 4: Rendered the Item List

I dynamically generated a list of items from the API data and displayed them on the page using DOM manipulation.

Step 5: Built the Detail View

To give users a deeper look at each item, I created a modal that appears when an item is clicked. This detail view was built using:

  • Event listeners to trigger the modal
  • A reusable modal component for consistent UI
  • Optional API calls to fetch extended data

Step 6: Styled the Interface

I used CSS to create a clean, responsive layout. The focus was on readability, spacing, and simple visual hierarchy.

Step 7: Ensured Code Quality

I formatted the JavaScript using ESLint/Prettier rules and tested the app to ensure there were no console errors or broken interactions.

Step 8: Deployed the App

Finally, I pushed the project to GitHub and deployed it using GitHub Pages so it could be publicly accessed.
