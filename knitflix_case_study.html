Knitflix - Case Study

Overview

Knitflix is a single-page web app where users can explore movies, directors, and genres — and build their own list of favorites. It’s fast, responsive, and built with the MERN stack (MongoDB, Express, React, Node.js).

Purpose and Context

I built Knitflix as part of of my Full Stack Web Development Course through Career Foundry to add to my professional portflio.

  • Create RESTful API that interacts with a database that stores data about different movies
  • Create several views (like movie details, profile, or favorites) using React
  • Demonstrate mastery of full-stack Javascript development.

Objective

Build the server-side and client-side for a movies web application. The movies application will allow users to access information about various movies, directors and genres. User will be able to sign up, update their profile and create a list of their favorite movies.

Server-Side Overview

This is the behind-the-scenes engine of Knitflix — the part that handles data, user accounts, and all the logic that powers the app. It’s built with Node.js, Express, and MongoDB, and deployed via Heroku. Here’s how I built it:

Step 1: Server Setup

I started by setting up Node.js, npm, and Express. Express acts as middleware — it’s a server-side framework that handles HTTP requests between the client and server. It’s what makes the app respond when users log in, browse movies, or update their profiles.

Step 2: Building the Database

I created a non-relational database using MongoDB, which is great for storing flexible movie data. I designed a schema — basically a blueprint — to define how movies and users would be structured. Then I seeded the database with: 10 sample movies 5 test users This gave me a solid foundation to test the app and build out features.

Step 3: Connecting API to Database

To connect the database to the API, I used Mongoose, a library that helps translate between Node.js and MongoDB. I wrote the business logic to handle things like: Fetching movie data Adding/removing favorites Updating user info Each API endpoint was designed to follow RESTful principles and return clean, consistent data.

Step 4: Authentication & Authorization

To keep user data safe, I added: Basic HTTP authentication for initial login JWT tokens for secure access to protected routes Passport.js to handle user sessions and token verification Once authenticated, users could safely interact with the API — like saving favorites or updating their profile.

Deployment & Testing

I deployed the API to Heroku, making it accessible to the React client. I used Postman throughout development to test endpoints, debug responses, and verify authentication flows.

Tools Used:

Server Framework Node.js, Express
Database MongoDB, Mongoose
Auth & Security Passport, JWT
Deployment Heroku
Testing Postman
